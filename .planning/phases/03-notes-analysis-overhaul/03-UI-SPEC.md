| Destructive | `--color-error: #ef5350` | `#ef5350` | Wrong answer annotations in marked-script, error percentage bars <60%, drill failure message |

**Accent reserved for:**

2. Drill generation progress spinner/indicator (`.drill-generator__progress`)

**Not accent:** Annotation highlights (use success/warning/error), error pattern bars (use success/warning/error based on percentage), drill card backgrounds (use secondary surface), marked-script passage text (use dominant/passage color).

- Part B1: `#ff9800` (hardcoded, matches existing)
- Part B2: `#e91e63` (hardcoded, matches existing)

## Copywriting Contract

| Element | Copy | Context |
|---------|------|---------|
| Primary CTA | `Generate Targeted Practice` | Drill trigger button in results view. Calls AI to create 2-3 focused questions. |
| Empty state (perfect score) | `No errors to review` | Shown when student scored 100%. Marked-script view shows passage without annotations. |
| Empty state (no drills yet) | `Review your mistakes above, then generate practice questions targeting your weak areas.` | Shown in drill section before any drills are generated. |
| Drill loading state | `Generating targeted practice questions...` | Shown while AI generates drills. Accompanied by spinner (`⏳`). |
| Drill ready state | `Practice these questions` | Section heading once drills are ready. |
| Drill fail state | `Practice questions could not be generated. The AI service may be unavailable. Try again or start a new session.` | Shown if AI generation fails or times out. Includes a "Try Again" button. |
| Drill answer reveal | `Check Answer` | Button label on each drill question before student answers. Changes to `Show Result` after answering. |
| Error pattern heading | `Error Pattern Analysis` | Section heading for the skill/type breakdown panel. |
| Marked script heading | `Annotated Passage` | Section heading for the marked-script view. |
| Weak area label | `Your weakest skill: {skill_name} ({percentage}%)` | Auto-generated label identifying the skill with lowest performance. |
| Drill section heading | `Targeted Practice` | Section heading for the drill generator card. |
| Notes generated success | `Study notes saved to your notes` | Existing — keep as-is (line 744 of ReadingModule.jsx). |
| Notes generated failure | `Study notes could not be generated` | Existing — keep as-is (line 749 of ReadingModule.jsx). |
| Destructive actions | **None in this phase** | No destructive actions. Drills are answerable inline; no delete/reset operations. |

**DSE instruction language for results context:**
- Section badge labels: `Part A — Compulsory`, `Part B1 — Easier (max Level 4)`, `Part B2 — Harder (up to 5**)`
- These match the existing part labels from ReadingModule.jsx
- No new DSE instruction language needed (results view is post-exam, not exam instructions)

## Interaction States

### MarkedScriptView States

| State | Condition | Visual |
|-------|-----------|--------|
| Loading | Passage HTML not yet parsed | Paragraph skeleton placeholders |
| No errors | All questions correct | Passage rendered without highlights. Banner: "No errors to review" in `.reading__results-source` area |
| Has errors | Any question wrong/partial | Paragraphs with errors highlighted with left-border accent (red). Annotation gutter shows per-question markers |
| Empty | No passage content | Hidden (don't render) — fall through to standalone error analysis |

### ErrorPatternAnalysis States

| State | Condition | Visual |
|-------|-----------|--------|
| Loaded | Questions with skillTested field exist | 3 card panels: Skill, Type, Weak areas |
| Empty | No questions with skillTested | Omit skill section. Show one panel: "Breakdown by Section" |
| Single section | Only one part | Skip section breakdown, focus on skill + type |

### DrillGenerator States

| State | Condition | Visual | Copy |
|-------|-----------|--------|------|
| `idle` | Not yet triggered | CTA card with primary button | "Review your mistakes above, then generate practice questions targeting your weak areas." + [Generate Targeted Practice] button |
| `generating` | AI call in progress | Disabled button + spinner | "⏳ Generating targeted practice questions..." |
| `ready` | Drills validated successfully | 2-3 QuestionRenderer instances stacked | "Practice these questions" heading |
| `answering` | Student answering drills | QuestionRenderer with `showResult: false`, per-question [Check Answer] button | Student answers inline, clicks check → reveals result |
| `answered` | All drills answered | QuestionRenderer with `showResult: true`, [Try Again] button | Shows correct/wrong per drill |
| `failed` | AI generation failed or validation failed | Card with error message + retry button | "Practice questions could not be generated. Try again or start a new session." + [Try Again] |

## Accessibility

- All annotation markers (✓/✗) MUST have `aria-label` equivalents (e.g., `<span aria-label="Correct">✓</span>`)
- Color is never the sole indicator — annotations include text labels ("Correct", "Wrong"), not just color
- The marked-script gutter uses enough contrast: success green on white (≥4.5:1 ratio with existing tokens)
- Drill buttons have focus-visible outline (already handled by existing CSS: `:focus-visible` rule at line 2795)
- All collapsible sections use `<details>` / `<summary>` (native HTML, accessible by default)
- Per-question results in the annotation gutter are links to the corresponding review item below
