  <action>
    Create `src/utils/errorPatternAnalysis.js` as a pure-function utility module following the exact pattern of `dseGrading.js`:

    **Module structure:**
    - Module-level constant `SKILL_LABELS` map (exported — for use by ErrorPatternAnalysis.jsx)
    - Module-level constant `SKILL_RECOMMENDATIONS` map (one recommendation string per skill)
    - Five named exports (no default export), all pure functions

    **SKILL_LABELS** (exported constant):
    ```js
    export const SKILL_LABELS = {
      mainIdea: 'Main Idea',
      detail: 'Detail Retrieval',
      inference: 'Inference',
      vocabInContext: 'Vocabulary in Context',
      tone: 'Tone & Attitude',
      purpose: 'Purpose',
    };
    ```

    **`analyzeBySkill(questions, answers)`** — REQUIRED by ErrorPatternAnalysis.jsx:
    - Import `checkAnswer` from `../utils/answerChecking`
    - Iterate over all questions, grouping by `q.skillTested` (default 'other' if missing)
    - For each skill: sum `q.marks || 1` as total; sum `checkAnswer(q, answers?.[q.id]).marksEarned` as earned
    - Return array sorted ascending by percentage (weakest first). Each element: `{ skill, label: SKILL_LABELS[skill] || skill, percentage: Math.round((earned/total)*100), total, earned, count }`
    - If no questions have skillTested, return empty array `[]`

    **`analyzeByType(questions, answers)`** — REQUIRED:
    - Group by `q.type`; same aggregation pattern as analyzeBySkill
    - Sort weakest-first. Use readable labels: `{ type, label: TYPE_LABELS[q.type] || q.type, percentage, total, earned, count }`
    - Define a local TYPE_LABELS map (not exported — same as QuestionRenderer's TYPE_LABELS)

    **`analyzeByPart(questions, answers, sections)`** — OPTIONAL (used by ErrorPatternAnalysis if sections exist):
    - Group questions by `q.part` (A/B1/B2)
    - For each part: compute total marks, earned marks, percentage
    - Return array: `[{ part, total, earned, percentage }]`

    **`identifyWeakAreas(bySkill, byType)`** — REQUIRED:
    - Takes the output of analyzeBySkill and analyzeByType
    - Returns flat array of weak area objects (percentage < 60 triggers inclusion)
    - Each object: `{ area, type: 'skill'|'type', percentage, severity: percentage < 40 ? 'critical' : 'needs-work', recommendation }`
    - Use SKILL_RECOMMENDATIONS for skill recommendations, generic text for type recommendations

    **`calculateSkillGap(bySkill, targetPercentage = 70)`** — OPTIONAL:
    - Filters skills below targetPercentage
    - Returns `[{ skill, gap, priority: gap > 30 ? 'high' : 'medium' }]`

    **Imports:** only `import { checkAnswer } from './answerChecking';`
    **Error handling:** Use guard clause: if no questions, return `[]` from all functions. No try/catch needed — pure functions.

  <name>Task 2: Create MarkedScriptView.jsx component</name>
  <files>src/components/MarkedScriptView.jsx</files>
  <behavior>
    - Renders passage HTML with per-paragraph highlights (green all-correct, red has-errors, amber partial)
    - Shows margin annotations in a right gutter: ✓/✗ per question, marks, feedback text
    - Falls back to proportional paragraph distribution when paragraphRef is missing
    - Empty state: "No errors to review" banner when all questions correct
    - Handles null/undefined passageHtml by returning null
    - Annotations use aria-label for accessibility (✓ → "Correct", ✗ → "Wrong")
  </behavior>
  <action>
    Create `src/components/MarkedScriptView.jsx` following patterns from ReadingModule.jsx (passage render lines 436-443) and MCQQuestion.jsx (correct/wrong indicators):

    **Imports:**
    ```js
    import React, { useMemo } from 'react';
    import { checkAnswer } from '../utils/answerChecking';
    ```

    **Props interface:**
    ```js
    {
      passageHtml: string,       // Raw HTML of the passage
      questions: Array,          // Full question array (with paragraphRef, marks, etc.)
      userAnswers: Object,       // Map of questionId -> user answer value
    }
    ```

    **Component logic:**

    1. **Parse passage paragraphs** (useMemo):
       - Use `document.createElement('div')` to parse HTML (not regex — per RESEARCH.md anti-pattern warning)
       - Filter to elements with tagName P, H2, H3, BLOCKQUOTE — all children that could be paragraph targets
       - Return `[{ index: number, html: string, tagName: string }]` for each element
       - Guard: if no passageHtml, return empty array

    2. **Map questions to paragraphs** (useMemo):
       - Build a `paragraphMap: { [ref: number]: Array<questionWithResult> }`
       - For each question: use `q.paragraphRef` if integer ≥ 1 ≤ totalParagraphs
       - Fallback: `Math.ceil((questions.indexOf(q) + 1) / Math.max(1, Math.floor(questions.length / Math.max(1, totalParagraphs))))` — proportional distribution
       - Call `checkAnswer(q, userAnswers[q.id])` to get per-question result
       - Store `{ ...q, result }` where result is the checkAnswer return

    3. **Render structure:**
       - Outer wrapper: `<div className="marked-script">`
       - If ALL questions correct: show a banner div `marked-script__empty` with text "No errors to review" (per UI-SPEC copy)
       - If any errors: render paragraphs as annotated rows
       
       Per paragraph row:
       ```
       <div className="marked-script__para [--has-errors|--all-correct]">
         <div className="marked-script__gutter">
           {annotations.map(q => (
             <div className="marked-script__annotation [--correct|--wrong|--partial]">
               <span className="marked-script__annotation-marker" aria-label={correct ? "Correct" : "Wrong"}>
                 {correct ? '\u2713' : '\u2717'}
               </span>
               <span className="marked-script__annotation-text">
                 Q{qNum} — {q.result.feedback}
               </span>
               <span className="marked-script__annotation-marks">
                 [{Math.round(q.result.marksEarned)}/{q.result.maxMarks}m]
               </span>
             </div>
           ))}
         </div>
         <div className="marked-script__text [--highlighted]">
           {/* dangerouslySetInnerHTML with para.html */}
         </div>
       </div>
       ```

    4. **CSS class toggling:**
       - `marked-script__para--has-errors`: any annotation with `!result.correct` and userAnswer non-null
       - `marked-script__para--all-correct`: all annotations correct
       - `marked-script__text--highlighted`: paragraph has any wrong answers
       - Annotation modifier: `--correct`, `--wrong`, `--partial` based on result.correct and marksEarned
       - `--partial`: when `!result.correct` but `marksEarned > 0`

    5. **Error handling:**
       - Guard clause: `if (!passageHtml || !questions?.length) return null;`
       - Guard clause: `if (!userAnswers) userAnswers = {};`

    6. **Accessibility** (per UI-SPEC):
       - All ✓/✗ markers need `aria-label`
       - Color is never the sole indicator — annotation shows text labels

    **Do NOT add any CSS** — CSS will be added in plan 03-03 to App.css. Use only the BEM class names from UI-SPEC.

    **Do NOT use `React.memo`** — the parent will handle re-render optimization.

    **Test at the end:** Open a browser console, mock some data and verify the component renders without errors.
