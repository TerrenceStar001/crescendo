# Plan 12-02: Hooks — Forgetting Curve, Plan Adaptation, Exercise Generation

Create the forgetting curve schedule hook, extend study plan with review integration, and add constraint injection to DSE papers AI pipeline.

## Tasks

### Task 1: Create useForgettingCurve hook
- State: reviewItems (loaded from DSE_KEYS.REVIEW_DATA), config (halfLife, threshold from localStorage)
- Key functions:
  - `getDueItems()` — returns items where recall < threshold
  - `recordReview(itemId, correct)` — updates half-life via calcHalfLife, stores new review date
  - `getRetrievability(itemId)` — returns current recall probability for an item
  - `updateConfig(newHalfLife, newThreshold)` — updates localStorage
- Persistence: load/save from IndexedDB

### Task 2: Extend useStudyPlan.js with review scheduling
- Add `getReviewSchedule()` function that returns items due for review from forgetting curve
- Add `getDailyPlan()` that merges study plan exercises + due review items
- This is the bridge between the study plan and forgetting curve

### Task 3: Add constraint injection to useDSEPapers.js
- The existing `callAI` pattern in useDSEPapers generates reading passages + questions
- Add an optional `constraints` parameter to the generation prompt
- When constraints are provided, inject them into the AI prompt as:
  "Generate a ${difficulty} difficulty ${type} question about ${theme} (format: ${format}, focus: ${focus})"
- This allows adaptive exercise generation that targets specific flaw categories
