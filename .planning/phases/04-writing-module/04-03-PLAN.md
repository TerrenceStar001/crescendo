    **`handleSubmit()` callback:**
    ```javascript
    const handleSubmit = useCallback(async () => {
      if (!partA.essay.trim() && !partB.essay.trim()) return;
      clearInterval(timerRef.current);
      setFocusMode(false);  // Exit distraction-free mode
      setPhase('correcting');
      setSubmitting(true);

      try {
        // Build correction prompt using the template from Plan 04-01
        const prompt = buildCorrectionPrompt(
          { essay: partA.essay, prompt: sessionData?.partA?.prompt },
          { essay: partB.essay, prompt: sessionData?.partB?.options?.[partB.chosenOption] },
          selfAssessment
        );

        const data = await callAI(prompt, {
          system: 'You are an expert HKDSE English examiner (Paper 2 Writing). Assess using official HKEAA criteria: Content/7, Organization/7, Language/7 per part. Return ONLY valid JSON.',
          temperature: 0.3,
          maxTokens: 2500,  // Larger token budget for 2-part correction
        });

        const parsed = parseCorrectionResponse(data);
        if (!parsed) throw new Error('Failed to parse AI response');

        // Assign DSE level using dseGrading.js (D-13)
        const dseLevel = scoreToDseLevel(parsed.overall.percentage, 'writing');
        parsed.overall.dseLevel = dseLevel.level;

        setCorrectionResult(parsed);
        setPhase('correction');

        // Save to IndexedDB writing sessions (D-35)
        saveSessionToHistory(parsed);

        // Record to skillAnalytics
        if (skillAnalytics) {
          skillAnalytics.recordSession({
            skill: 'writing',
            type: 'practice',
            score: parsed.overall.total,
            totalQuestions: parsed.overall.maxTotal || 42,  // 21 per part × 2
            percentage: parsed.overall.percentage,
            subScores: {
              content: ((parsed.content?.score || 0) / 7) * 100,
              organization: ((parsed.organization?.score || 0) / 7) * 100,
              language: ((parsed.language?.score || 0) / 7) * 100,
            },
            wordCount: (partA.wordCount || 0) + (partB.wordCount || 0),
            duration: sessionData?.duration || 7200 - (timeRemaining || 0),
            dseLevel: dseLevel.level,
          });
        }

        // Clear session storage (session complete)
        clearSessionStorage();
      } catch (e) {
        console.error('Failed to correct essay:', e);
        setCorrectionResult({
          content: { score: 0, feedback: 'Correction failed. Please try again.' },
          organization: { score: 0, feedback: '' },
          language: { score: 0, feedback: '' },
          overall: { total: 0, maxTotal: 42, percentage: 0, dseLevel: '—', narrativeSummary: 'The AI examiner encountered an error. You can try submitting again.' },
          errors: [],
          vocabularySuggestions: [],
          goodLanguage: [],
          sectionBreakdown: {},
          pitfallsAvoided: [],
          inlineAnnotations: [],
        });
        setPhase('correction');
      } finally {
        setSubmitting(false);
      }
    }, [partA, partB, sessionData, selfAssessment, callAI, skillAnalytics, timeRemaining, scoreToDseLevel]);
    ```

    - Limits history to 50 entries (per RESEARCH risk mitigation)
    - Writes back with `setItem`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI response → frontend | AI returns JSON with score/error data; malformed or malicious JSON handled by parseCorrectionResponse |
| IndexedDB write | Session history persisted client-side; no server exposure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Tampering | AI response JSON | mitigate | parseCorrectionResponse() validates scores 0-7, errors is array, narrativeSummary is string, falls back to default values on null |
| T-04-08 | Spoofing | IndexedDB session history | accept | Client-side only, no PII expected in practice essays |
| T-04-09 | Tampering | Self-assessment tags | mitigate | Tags are simple string arrays, validated by buildCorrectionPrompt |
| T-04-10 | Tampering | AI prompt injection via essay text | mitigate | System prompt is prepended and temperature=0.3 reduces variability; no user data executed as code (per RESEARCH §Security Domain) |
| T-04-11 | Spoofing | XSS via rendered AI correction JSON | mitigate | Render AI feedback text via textContent or React JSX (auto-escaped), never dangerouslySetInnerHTML. AI returns `original`/`explanation` strings — render as text only. |
</threat_model>
