<objective>
Create a shared semantic validation engine and align hyper-parameters across all course generation paths. The semantic validator checks content validity (MCQ answers in options, gap-fill answers in content, explanation length, reference content depth, exercise volume) and integrates into the retry loop to inject BOTH structural AND semantic errors back into the prompt.

Purpose: Fix the root cause of hollow courses — AI generates structurally valid but semantically invalid content (e.g., MCQ correctAnswer not in options, gap-fill answers not in passage). The shared validator prevents drift between server and client validation logic.
Output: `server/utils/courseSemanticValidator.js`, modified `server/routes/courses.js`, modified `src/utils/courseSchema.js`
</objective>


    **D) Improve `parseJSONResponse()`** (D-29, lines 120-130):
    Replace naive single-brace-match with multi-strategy approach:
    1. Try to find the first `{` and last `}` — attempt JSON.parse on that substring
    2. If that fails, try progressive nesting: find the first `{`, then iterate forward tracking brace depth until depth returns to 0 — parse that substring
    3. Try regex match for a complete JSON: `/\{(?:[^{}]|(?:\{(?:[^{}]|(?:\{[^{}]*\}))*\}))*\}/g` — find all objects and try each
    4. If all fail, return null (caller handles via retry loop)

    All four strategies MUST be attempted before returning null. No silent parse failures.

    The `POST /api/courses/auto-generate` endpoint uses `callAICourse()` which now has the correct params. The `PUT /ingest/generate/:extractionId` also uses `callAICourse()`. Both paths automatically benefit from the change.
