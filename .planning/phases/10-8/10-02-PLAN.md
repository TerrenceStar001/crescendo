    (Per D-14, D-15, D-16, D-17, D-18, D-19) Create `src/context/CourseGenerationContext.jsx` following the ViewContext.jsx pattern. Must use `createContext`, `useContext`, `useState`, `useCallback`, `useRef`, `useEffect` from React.

    **A) State shape:**
    ```javascript
    const [state, setState] = useState({
      isGenerating: false,
      progress: 0,
      status: 'idle', // 'idle' | 'generating' | 'success' | 'error'
      error: null,
      courseId: null,
      courseTitle: null,
    });
    ```

    **B) Pseudo-progress engine** (D-15):
    Use `setInterval` with 500ms granularity. Track start time with `useRef`. Three phases:
    - Phase 1 (0-30s): `progress = Math.min(45, (elapsed / 30000) * 45)`
    - Phase 2 (31-90s): `progress = 45 + ((elapsed - 30000) / 60000) * 40` → targets 85%
    - Phase 3 (91-115s): `progress = 85 + ((elapsed - 90000) / 25000) * 10` → targets 95%
    - Phase 4: On explicit `completeGeneration()` call → set progress to 100
    Clear interval on `stopGeneration()`, `cancelGeneration()`, or when status leaves 'generating'.

    **C) Generation lifecycle methods:**
    - `startGeneration({ weaknessTags, completedCourseIds, callAI, autoGenerateCourseFn, simplerContent })`:
      Sets status='generating', starts progress interval, calls `autoGenerateCourseFn(weaknessTags, completedCourseIds, callAI, { simplerContent })`.
      On success: sets status='success', courseId, courseTitle, progress=100, clears interval.
      On failure: sets status='error', error=message, sets progress to current value (stops), clears interval.
    - `retryGeneration()`: Calls `startGeneration()` with same original params.
    - `retrySimpler()`: Calls `startGeneration()` with `simplerContent: true`. This reduces AI prompt scope to 1 topic, 2 exercises/lesson, and appends prompt modifier: `"CRITICAL: Generate highly concise, straightforward reference passages (150-200 words max) and prioritize highly predictable, standardized question syntax to maximize validation compliance."`
    - `cancelGeneration()`: Sets status='idle', clears progress interval.
    - `dismiss()`: Resets to idle state.

    **D) Navigation guard** (D-16):
    Use `useEffect` with `window.addEventListener('beforeunload', handler)`:
    - When `isGenerating`, attach `beforeunload` handler that sets `event.preventDefault()` and `event.returnValue = ''`
    - On cleanup, remove the listener

    For React Router nav guard: render a custom modal overlay when `isGenerating` and user attempts to navigate (check via state change). The modal shows: "Your course is generating in the background." with buttons: "Browse Site" (dismisses guard, lets user browse, generation continues in background) and "Stay on Page" (closes modal).

    **E) Floating panel UI** (D-15, D-18, D-19):
    Render a fixed-position bottom-right panel when status !== 'idle':
    - **Generating**: Blue/neutral panel with progress bar, status text "Generating your course...", cancel button (X)
    - **Success** (D-18): Green panel with checkmark, text "Your personalized course is ready!", and "[View Course]" link/button that navigates to the course in catalog
    - **Error** (D-19): Red panel with error icon, text with error message, "Retry" and "Try Simpler Content" buttons, "Dismiss" link

    Panel CSS classes: `course__gen-panel`, `course__gen-panel--generating`, `course__gen-panel--success`, `course__gen-panel--error`
    Progress bar: `course__gen-progress-bar` with width from state.progress + `%`

    **F) beforeunload behavior** (D-17):
    On tab close during generation: the interval stops (tab hidden). On next visit, check localStorage for a pending generation key. If found, show success toast. Implementation: store pending generation info in `sessionStorage` key `crescendo-gen-pending` with `{ courseId, courseTitle }` before starting. On success, clear it. On next app load, check for this key and show a toast if present.

    **G) Expose via context:**

    ```javascript

    export function CourseGenerationProvider({ children, autoGenerateCourseFn, callAI }) { ... }
    export function useCourseGeneration() { ... }
    ```

    The provider accepts `autoGenerateCourseFn` and `callAI` as props (provided by App.jsx) so the context can invoke generation directly without App having to orchestrate it.
