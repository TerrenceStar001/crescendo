  <action>
  Add a new POST endpoint `/api/rag/fragments` to `server/index.js` after the existing POST `/api/rag/search` handler (line 371).

  Endpoint specification:
  - Accepts `{ topic, difficulty, count=3, fragmentMaxWords=300 }` from request body
  - If `ragEngine` is null, return 503 `{ error: 'RAG engine not available' }`
  - If no `topic` provided, default to `'feature'`
  - Call `ragEngine.search(topic, count * 2)` to get candidate chunks
  - For each result chunk, extract the source article ID (strip `-\d+$` suffix from chunk id)
  - Fetch full article from the `articles` table using `db.prepare('SELECT id, title, content, source, date, word_count FROM articles WHERE id = ?')`
  - Skip articles with content length < 200 characters
  - For each valid article: strip HTML tags, split into paragraphs on `\n\n+`, filter paragraphs with length > 50 chars
  - Score paragraphs by query term density (count how many words from the topic string appear in each paragraph), take the top 1-2 paragraphs per article
  - Limit fragment text to `fragmentMaxWords` words per article
  - Build `sourceName` mapping: `'scmp' → 'South China Morning Post'`, `'youth-post' → 'Young Post'`, default to the source value
  - Format `sourceDate` using `toLocaleDateString('en-US', { year: 'numeric', month: 'long' })`
  - Return JSON `{ fragments: Array<{ text, sourceName, sourceDate, sourceId }>, topic, difficulty }`
  - Wrap in try/catch — errors return 500 with `{ error: e.message }`
  - Follow the same error handling and middleware pattern as the existing `/api/rag/search` endpoint (lines 329-371)

  No new imports. No middleware changes. The `ragEngine` and `db` variables are already available at module scope (lines 25, 28).
  </action>

  <acceptance_criteria>
  - `curl -X POST http://localhost:3001/api/rag/fragments -H 'Content-Type: application/json' -d '{"topic":"education","count":2}'` returns 200 with a `fragments` array
  - Each fragment has `text`, `sourceName`, `sourceDate`, `sourceId` fields
  - Fragment text is ≤ 300 words (fragmentMaxWords default)
  - Re-running with `ragEngine = null` condition returns 503 with `{ error: 'RAG engine not available' }`
  - Missing `topic` defaults to `'feature'`
  </acceptance_criteria>
