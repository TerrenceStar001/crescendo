## Spacing Scale

Existing 8-point scale (multiples of 4) confirmed from `src/App.css`: `--space-1`(4px) through `--space-8`(48px). No spacing tokens beyond `--space-8` exist. No changes needed.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Inline padding, tight gaps |
| `--space-2` | 8px | Compact element spacing |
| `--space-3` | 12px | Paragraph margin-bottom (existing, to be replaced) |
| `--space-4` | 16px | Default element spacing, passage column padding |
| `--space-5` | 24px | Section padding, passage card margin |
| `--space-6` | 32px | Layout gaps |
| `--space-8` | 48px | Major section breaks |

**Exceptions:**
- First-line indent on passage paragraphs: **2em** (32px at 16px base) ŌĆö not a spacing token, set via `text-indent`
- Passage card padding: use `var(--space-5)` (24px) for top/bottom, `var(--space-6)` (32px) for left/right to simulate booklet margins
- Line number gutter: 40px minimum width, fixed, not using spacing tokens

### Passage Body Typography (new contract)
| Role | Size | Weight | Line Height | Font Stack | Alignment |
|------|------|--------|-------------|------------|-----------|
| Passage body | 1rem (16px) | 400 (regular) | 1.6 | Georgia, 'Times New Roman', serif | Justified |
| Passage title | 1.15rem (18.4px) | 700 (bold) | 1.3 | Georgia, 'Times New Roman', serif | Left |
| Text header (Text 1/2) | 0.9rem (14.4px) | 700 (bold) | 1.3 | system-ui, -apple-system, sans-serif | Left |
| Part header (Part A/B1/B2) | 0.85rem (13.6px) | 700 (bold) | 1.3 | system-ui, -apple-system, sans-serif | Left |
| Source attribution | 0.8rem (12.8px) | 400 (regular), italic | 1.4 | Georgia, 'Times New Roman', serif | Right |
| Line numbers | 0.7rem (11.2px) | 400 (regular) | 1.6 | system-ui, -apple-system, sans-serif | Right-aligned in gutter |

**Rationale:** DSE Paper 1 booklets use serif fonts for passage body text (Times New Roman ŌĆö standard for British exam boards). The existing sans-serif system font is appropriate for UI chrome (headers, badges, navigation) but the passage itself should use serif for authenticity. 16px body text maps to 12pt print equivalent (standard DSE booklet size). Justified text mirrors print booklet composition.

### Structural HTML from AI Generator (output format contract)

The AI generator in `useDSEPapers.js` produces passage HTML. This format is already partially implemented. The contract formalizes it and adds structural requirements:

```html
<!-- Single-text passage -->
<h2>Passage Title</h2>
<p>First paragraph content...</p>
<p>Second paragraph content...</p>
<p class="reading__attr-source">[Attribution footer ŌĆö added by post-processor, not AI]</p>

<!-- Multi-text passage -->
<h2>Overall Passage Title</h2>
<h3>Text 1</h3>
<p>Text 1, paragraph one...</p>
<p>Text 1, paragraph two...</p>
<h3>Text 2</h3>
<p>Text 2, paragraph one...</p>
<p class="reading__attr-source">[Attribution footer ŌĆö added by post-processor, not AI]</p>
```

**Requirements:**
- AI must NOT output `<p class="reading__attr-source">` ŌĆö this is added client-side from metadata
- AI must NOT output line numbers ŌĆö these are generated client-side via CSS counters
- AI must NOT output `<br>` line breaks within paragraphs ŌĆö use proper `<p>` tags
- AI output remains `<h2>`, `<h3>`, `<p>` only ŌĆö no additional elements
