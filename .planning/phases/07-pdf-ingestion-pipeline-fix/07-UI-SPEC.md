## Component Inventory

### New UI Elements (Phase 7)

These are the new visual/interaction contracts this phase introduces into `CourseIngestion.jsx` and the catalog:

#### 1. QualityPreviewScreen

**State machine position:** Inserted as new phase `'quality'` between `'parsing'` (text extraction done) and `'generating'` (AI structuring starts). Flow: `idle → uploading → parsing → **quality** → generating → review → saving → done`.

**Purpose:** Shows extraction quality metrics and lets user decide whether to proceed with AI structuring.

**Visual contract:**
- Container: `course__ingestion` (same as other phases)
- Card: `course__quality-preview` — white surface card with border, `--space-5` padding
- Title area: `course__quality-header` — `display: flex; align-items: center; gap: --space-3`
  - Score badge: `course__quality-score` — circular badge, `--font-2xl` weight 700
    - Color: `--color-success` when score is PASS (≥500 chars AND ≥70% English)
    - Color: `--color-warning` when score is MARGINAL (meets one threshold but not both — only if pdf2md fallback passes)
    - Color: `--color-error` when score is FAIL (otherwise)
  - Title + subtitle stacked vertically
- Table: `course__quality-table` — per-page breakdown
  - Header row: `course__quality-table-header` — Page, Characters, English %, Status
  - Body rows: `course__quality-table-row`
    - Status indicator: full-width green bar (relative width = char count / max page), or red X if <100 chars
  - Totals row: `course__quality-table-total` — bold, bottom border
- Actions: `course__quality-actions` — flex row with `justify-content: space-between`
  - "Cancel" button: `course__btn course__btn--secondary` — returns to idle upload
  - "Proceed to Course Draft" button: `course__btn course__btn--primary` — only enabled when quality is PASS or MARGINAL
- Full-width block banner `course__quality-block` when FAIL:
  - Background: `--color-error-bg`
  - Border: 1px solid `--color-error`
  - Heading: "Insufficient Content" — `--font-base` weight 600, `--color-error`
  - Body: explanation with dynamic char count and English %
  - "Try a Different File" button: `course__btn course__btn--primary` with `--color-error` accent

**Interaction contract:**
- "Proceed to Course Draft" triggers `setPhase('generating')` → AI structuring call
- "Cancel" triggers full reset: `handleDiscard()` equivalent
- "Try a Different File" returns to `idle` phase, clears file
- FAIL state **must not** allow proceed — "Proceed" button hidden or disabled

**States:**
| State | Trigger | Visual |
|-------|---------|--------|
| PASS | ≥500 chars AND ≥70% English | Green score badge, proceed button enabled |
| FAIL | <500 chars OR <70% English | Red block banner, no proceed option |
| MARGINAL | pdf2md fallback succeeds where primary fails | Yellow score badge, proceed enabled with warning toast |

#### 2. EnhancedErrorBanner

**Purpose:** Replace the existing single `course__error-msg` pattern with role-specific banners that have clear visual hierarchy.

**Where it appears:**
- In upload form (`renderIdle()`)
- In quality preview screen (`renderQuality()`)
- In review screen (`renderReview()`) — already exists

**Visual contract:**
- Container: `course__error-banner` — full-width block
  - Icon area (left): `course__error-banner-icon` — "⚠" or custom SVG (16×16)
  - Content area (middle):
    - `course__error-banner-title` — bold label (e.g. "Upload Failed", "Insufficient Content")
    - `course__error-banner-body` — description text, `--font-sm`
  - Action area (right): `course__error-banner-action` — retry button
- Background: `--color-error-bg`
- Border-left: 3px solid `--color-error` (colored accent bar, like existing `.note-card--error` pattern if it exists, or new)
- Border: 1px solid `--color-error`
- Border-radius: `--radius`

**Interaction contract:**
- Error banner has built-in "Try Again" button with distinct visual
- Clicking "Try Again" resets the relevant phase and re-opens file picker
- Error banner does NOT auto-dismiss — user must take action

**Error variants:**

| Error Type | Title | Body | Action |
|-----------|-------|------|--------|
| File type | "Invalid File Type" | "Please select a .pdf file." | — (part of dropzone, not a banner) |
| File size | "File Too Large" | "File exceeds the 10MB limit. Please choose a smaller file." | "Try Again" → opens file picker |
| Upload failure | "Upload Failed" | `data.error` or "Network error. Please check your connection and try again." | "Try Again" → re-uploads |
| Extraction failure | "Extraction Failed" | "Could not read text from this PDF. Try a file with more text content." | "Try a Different File" → returns to idle |
| Quality fail | "Insufficient Content" | "{chars} characters, {engPct}% English. Needs ≥500 chars and ≥70% English." | "Try a Different File" → returns to idle |

#### 3. ToastNotification

**Purpose:** Non-blocking notification for minor issues (partial page extraction, sync success).

**Pattern:** Reuse `.undo-toast` pattern from `src/App.css` line 1459.

**Visual contract:**
- Position: fixed, bottom `--space-5`, centered horizontally, z-index `9999`
- Container: `course__toast` (extends `.undo-toast` base)
  - No action button (unlike undo-toast)
  - Single line of text, `--font-sm`, `--color-text-secondary`
  - Optional: leading icon
- Background: `--color-warning-bg` for warnings, `--color-success-bg` for success
- Border: 1px solid `--color-warning` / `--color-success`
- Animation: `toast-in 0.3s ease-out` (reuse existing keyframe at line 2798)
- Auto-dismiss after 4s or on click/tap outside

**Variants:**

| Type | Background | Icon | Example Message |
|------|-----------|------|-----------------|
| Warning | `--color-warning-bg` + `--color-warning` border | ⚠ | "Some pages had limited text (3 pages with <100 chars)" |
| Success | `--color-success-bg` + `--color-success` border | ✓ | "Courses synced from server (12 courses)" |

#### 4. RefreshCoursesButton

**Purpose:** Manual IndexedDB sync trigger in the catalog view.

**Location:** `CourseOverview.jsx` — in the catalog header area, alongside search bar or section title.

**Visual contract:**
- Button: `course__btn course__btn--refresh`
  - Size: compact (same as tag chips, ~28px height)
  - Icon: inline SVG (refresh arrows), 14×14, `--color-text-muted`
  - Text: "Refresh Courses" — `--font-xs` weight 600
  - Background: transparent, border: `1px solid --color-border`
  - Hover: `--color-surface-hover` background
- Disabled state: `course__btn--disabled` — `opacity: 0.5`, cursor not-allowed
  - Applies when offline (no `navigator.onLine`)
- Loading state: spinner icon replaces arrow text, "Syncing..." label

**Interaction contract:**
- Click triggers `fetch('/api/courses/sync')` → stores response in IndexedDB
- Shows loading state during fetch (spinner + "Syncing...")
- On success: toast notification "Courses synced from server ({N} courses)"
- On failure: error banner "Sync failed — {error.message}"
- Disabled when `navigator.onLine === false`
- No auto-sync on page load (per D-12)

## Interactions

### State Machine (Updated)

```
idle
  → user selects/drops PDF
  → uploading

uploading
  → client-side validation (type + size)
  → validation fails → idle (with error banner)
  → validation passes → parsing

parsing
  → backend /api/courses/ingest called
  → extraction quality returned
  → quality

quality (NEW)
  → user reviews per-page stats
  → "Cancel" → idle
  → quality FAIL → idle (with error banner)
  → quality PASS/MARGINAL + "Proceed" → generating

generating (existing)
  → AI structuring on backend
  → success → review
  → failure → idle (with error banner)

review (existing)
  → user edits + "Publish" → saving
  → "Discard" → idle

saving (existing)
  → /api/courses/{id}/publish called
  → success → done
  → failure → review (with error banner)

done (existing)
  → "View in Catalog" → back to catalog
  → "Upload Another" → idle
```
