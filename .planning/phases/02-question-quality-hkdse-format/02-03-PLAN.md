  <action>
    Create `src/utils/questionGenerator.js`. Export prompt builder functions that return string fragments. Each function receives optional context (passage preview, distribution targets, part) and returns a string prompt section.

    Required exports:

    1. **`buildMCQPrompt(numQuestions, part)`** — Returns prompt section for MCQ generation:
       - Stem MUST be a question ending with `?`
       - Exactly 4 labelled options (A/B/C/D)
       - Only ONE correct option
       - Each distractor targets one of 5 cognitive trap types (extreme-wording / similar-sounding / partial-truth / out-of-scope / numerical)
       - Opposite traps explicitly banned
       - References `COGNITIVE_TRAP_TYPES` from questionTypes.js
       - `< 60% word overlap between any two options`

    2. **`buildTFNGPrompt(numQuestions)`** — Returns prompt section for TFNG:
       - Stem MUST be a declarative STATEMENT ending with `.`
       - correctAnswer exactly "T", "F", or "NG" (single letter)
       - FALSE requires explicit verifiable contradiction in text
       - NOT GIVEN: plausible but unverifiable from text
       - If ≥4 TFNG, at least 2 MUST be NG (constraint during generation, not post-processing, per D-05)
       - If 2-3 TFNG, at least 1 MUST be NG

    3. **`buildClozePrompt()`** — Returns prompt section for gap-fill/cloze:
       - Blank represented by `_____` or `{1}` in stem
       - answers array with correct words in order
       - Blanks draw from passage but word must NOT appear verbatim in adjacent text
       - Part of speech must match (noun→noun, verb→verb)

    4. **`buildShortAnswerPrompt()`** — Returns prompt section for short-answer:
       - Answer with word/phrase from passage (or own words if specified)
       - Include acceptableAnswers array for alternatives
       - wordLimit field when applicable
       - For questions with marks ≥2, include rubric with requiredPoints

    5. **`buildMatchingPrompt()`** — Returns prompt section for matching:
       - Minimum 3 pairs
       - REQUIRES pairs array AND options array
       - Column labels vary by skillTested (cause→effect, claim→evidence, term→definition)

    6. **`buildSummaryClozePrompt()`** — Returns prompt section for summary cloze:
       - Summary table covering multiple paragraphs
       - Numbered blanks (1, 2, 3...)
       - Each blank = ONE word from passage
       - answers array with correct words

    7. **`buildPronounRefPrompt()`** — Returns prompt section for reference questions:
       - "What does [word] in paragraph X refer to?"
       - Must specify line/paragraph reference
       - Include acceptableAnswers for pronoun antecedents

    8. **`buildSemanticConnectPrompt()`** — Returns prompt section for semantic matching:
       - Match causes to effects, or claims to evidence
       - REQUIRES pairs and options arrays
       - Minimum 3 pairs

    9. **`composeFullPrompt(passagePreview, numQuestions, part, typeDist)`** — Composes the full generation prompt by:
       - Including passage
       - Adding shared rules (paraphrase, truncation ban, conciseness, JSON format)
       - Adding per-type prompt sections based on typeDist distribution for the part
       - Returning the complete prompt string

    Use UPPER_SNAKE_CASE for format-level constants and camelCase for builder functions, matching the questionTypes.js / structuralConstraints.js pattern.

    Per D-08: Separate AI prompts for each question type + post-generation validation.
    Per D-05: TFNG NG constraint during generation (not post-processing).
    Per D-03: All HKDSE question types covered.
  </action>

    Required exports:

    1. **`validateQuestion(q)`** — Per-question validation:
       - Returns `{ valid, warnings, normalized }` where normalized is the question with fixes applied
       - Check stem exists and not truncated (use existing `detectTruncatedStem` logic)
       - Check correctAnswer exists and matches expected format per type:
         - mcq: single letter A-D
         - tfng: single letter T/F/NG
         - gap-fill: comma-separated words or answers array
         - short-answer: < 30 words
         - matching: pairs array + options array
         - summary-cloze: answers array with strings
         - pronoun-ref: single string
         - semantic-connect: pairs + options arrays
       - Check marks is integer 1-3
       - Check rubric exists for marks ≥2
       - Check explanation exists (warning, not error)
       - Wrap existing logic from `validateQuestionAnswer` (lines 241-281)

    2. **`validateDistractors(mcqQuestions)`** — Specific to MCQ type:
       - For each MCQ question, check options are 4 items with labels A/B/C/D
       - Check word overlap < 60% between any two options (same logic as lines 336-352)
       - Check no option is a direct negation of the correct answer
       - Return `{ valid, warnings }`

    3. **`validateTFNGDistribution(questions)`** — TFNG-specific check:
       - Same logic as ensureNGCount (lines 283-299) but as validation reporter
       - Returns `{ valid, warnings, needsFix }` — needsFix is boolean
       - If needsFix, the caller should regenerate (per D-05: no post-processing rebalance)

    4. **`validateTypeConsistency(questions)`** — Cross-type consistency:
       - Check at least 4 unique types used (same as lines 387-390)
       - Check paragraph distribution: ≥30% from second half
       - Check no stem is a direct passage copy
       - Check no truncation references in stem/explanation
       - Return `{ valid, warnings }`

    5. **`validateQuestions(questions, passagePlain)`** — Composite validator:
       - Runs validateQuestion on each question
       - Filters valid questions, collects warnings for each invalid
       - Runs validateDistractors on MCQ questions
       - Runs validateTFNGDistribution
       - Runs validateTypeConsistency
       - Returns `{ valid: overallBoolean, warnings: allWarnings[], questionResults: [{ id, valid, warnings }] }`

    All functions are pure. They do NOT mutate the questions array. They return results as new objects.

    Per D-08: Post-generation validation layer. Per D-05: TFNG NG distribution enforced via validation (reject if insufficient NG).
