  for (const q of questions) {
    const type = q.type || 'mcq';
    if (subScores[type] === undefined) { subScores[type] = 0; counts[type] = 0; }

    1. **`UK_US_MAP`** ŌĆö Object mapping UK spellings to US spellings (and vice versa via reverse lookup):
    ```javascript
    {

      'honour': 'honor', 'humour': 'humor', 'labour': 'labor', 'neighbour': 'neighbor',
    }
    ```

    2. **`normalizeAnswer(text)`** ŌĆö Returns normalized string with:
       - Trim whitespace
       - Convert to lowercase
       - Strip leading articles: remove `^a `, `^an `, `^the ` (word-initial only, each tried once)
       - Strip trailing articles / punctuation-adjacent
       - Normalize UKŌåÆUS spelling (swap UK to US variant per the map above)
       - Collapse multiple spaces to one
       - Return trimmed result

    3. **`checkAnswer(question, userAnswer)`** ŌĆö Returns `{ correct: boolean, marksEarned: number, maxMarks: number, feedback: string }`.
       - `maxMarks` = `question.marks || 1`
       - Switch on `question.type`:
         - **mcq**: `correct = normalizeAnswer(userAnswer) === normalizeAnswer(question.correctAnswer)`, marksEarned = maxMarks if correct else 0
         - **tfng**: normalize both to T/F/NG single letter, exact match, marksEarned = maxMarks if correct else 0
         - **gap-fill**: If `question.answers` (array), check each blank individually. Each blank worth `maxMarks / answers.length`. marksEarned = sum of correctly-matched blanks. correct = (marksEarned === maxMarks).
           If single answer: use normalizeAnswer match, marksEarned = maxMarks if correct else 0.
         - **matching**: Each pair worth `maxMarks / pairs.length`. marksEarned = sum of correctly-matched pairs. correct = (marksEarned === maxMarks).
         - **short-answer**: Check `question.acceptableAnswers` array first (if present, accept any). Then use normalizeAnswer matching. If no exact match, try keyword matching from existing logic (key terms from correctAnswer, ignore stopwords). For questions with `rubric.requiredPoints`, each required point = `maxMarks / requiredPoints.length`. marksEarned = points matched. correct = (marksEarned === maxMarks).
         - **open-ended**: Same as short-answer but always returns partial marks if rubric exists.
         - **summary-cloze**: Same as gap-fill multi-blank logic using `question.answers` array.
         - **pronoun-ref**: normalizeAnswer exact match, marksEarned = maxMarks if correct else 0.
         - **semantic-connect**: Same as matching ŌĆö each pair worth equal share of maxMarks.
       - `feedback`: "Correct!" / "Incorrect" / "Partially correct (X/Y marks)" for partial marking cases.
       - Return `null` for null/undefined/empty userAnswer.

    4. **`computeScore(questions, userAnswers)`** ŌĆö Convenience function:
       - Iterates questions, calls checkAnswer for each
       - Returns `{ marksEarned, maxMarks, percentage, results: [{ questionId, correct, marksEarned, maxMarks, feedback }] }`

    Per D-06: DSE-style rubric with partial marks and acceptable alternatives.
    Per D-07: Case + article tolerance (case-insensitive, ignore articles, UK/US variants). No Levenshtein fuzzy matching.
    Per D-04: 5 cognitive trap categories (already in questionTypes.js, referenced here for feedback).
