## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Course data → PWA cache | Cached course content served offline — must validate before rendering |
| Skill analytics → Course unlock | Difficulty progression based on DSE level — client-side only, not security-sensitive |
| Auto-generated courses → Student catalog | AI-generated content displayed to student — quality gate at generation time |
| PostTaskSuggestion → localStorage | Dismissal data stored in localStorage — no PII, no security concern |
| Re-generation trigger → ReadingModule/WritingModule | Adds course recommendation after task submission — modifies existing module behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-13 | Information Disclosure | Cached course content offline | mitigate | Cached courses validated via validateCourse() before rendering. No PII stored in cache. |
| T-06-14 | Tampering | Difficulty progression bypass (client-side) | accept | DSE level check is UI-only gating. Not a security boundary — students can access any course by navigating directly. Acceptable for MVP. |
| T-06-15 | Manipulation | Auto-generated course quality | mitigate | validateCourse() runs on AI response before saving. Invalid courses are discarded with retry. Quality gates at generation time per D-37. |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages. All existing packages approved. |
| T-06-16 | Spoofing | PostTaskSuggestion dismissals bypassed | accept | Dismissal is convenience UX, not security boundary. Users can dismiss/re-see at will. Acceptable for MVP. |
| T-06-17 | Information Disclosure | Recommendations reveal specific weakness patterns in post-task banner | mitigate | Recommendations shown only in the course recommendation banner text (tag-level, not raw error data). No specific error details disclosed. |
</threat_model>
