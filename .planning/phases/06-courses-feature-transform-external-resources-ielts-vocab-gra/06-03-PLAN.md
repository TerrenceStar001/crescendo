         - cloze: text with blanks, student fills in each blank
         - reordering: draggable list items to reorder into correct sequence
         - short-answer: textarea with word limit
         - mcq: radio button options

       - Auto-save (following WritingModule pattern):
         ```
         const saveTimerRef = useRef(null);
         useEffect(() => {
           if (phase === 'overview') return;
           clearTimeout(saveTimerRef.current);
           saveTimerRef.current = setTimeout(() => {
             try {
               dsePapers.setItem(COURSE_PROGRESS_KEY, JSON.stringify({
                 courseId: course.id,
                 topicIndex: currentTopicIndex,
                 lessonIndex: currentLessonIndex,
                 exerciseIndex: currentExerciseIndex,
                 answers,
                 referenceUnlocked,
                 savedAt: Date.now(),
               }));
             } catch {}
           }, 10000);
           return () => clearTimeout(saveTimerRef.current);
         }, [answers, phase, currentTopicIndex, currentLessonIndex, currentExerciseIndex]);
         ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CoursePlayer → IndexedDB | Progress data written to client storage — tamperable but user-local only |
| Course data → Exercise rendering | Course content rendered as JSX — must sanitize to prevent XSS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-10 | Tampering | Progress data in IndexedDB | mitigate | Course data validated on load via validateCourse(). Progress data structure enforced by schema. |
| T-06-11 | Information Disclosure | Exercise content rendered as JSX | mitigate | All course content rendered with React's built-in JSX escaping (dangerouslySetInnerHTML not used). Question text and explanations are treated as text content only. |
| T-06-12 | Manipulation | Reference content unlock bypass | accept | Reference content unlock is a learning UX feature, not a security boundary. Students can inspect DOM to see content — acceptable tradeoff for exercise-first pedagogy. |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages. All existing packages approved. |
</threat_model>
