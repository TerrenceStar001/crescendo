  <action>
Create `src/hooks/useFlawDetection.js` with default export function `useFlawDetection(sessions)`:

**Purpose:** Takes an array of DSE session records (from useSkillAnalytics), classifies errors using flawClassification.js, maintains a sliding window (7 days default), and provides aggregated flaw stats.

**State:**
- `flawRecords` — array of classified flaw objects loaded from IndexedDB (DSE_KEYS.FLAW_DATA)
- `isLoaded` — boolean flag

**Flaw record shape:**
```js
{ id: string, sessionId: string, skill: string, questionId: number,
  category: 'vocabulary'|'grammar'|'comprehension'|'inference'|'synthesis'|'expression',
  severity: 'micro'|'meso'|'macro',
  questionType: string, marksLost: number, confidence: number,
  timestamp: 'ISO string' }
```

**Key functions (all useCallback):**

1. `processSessions(sessionList)` — The core classification function:
   - Iterates over each session in `sessionList`
   - For each session, checks if it's already been processed (by sessionId stored in a Set)
   - For new sessions, extracts incorrect answers by comparing session.answers against session.questions
   - For each wrong answer:
     - Determines marksLost from question.marks
     - Calls `classifyFlaw(question.type, question.stem, marksLost)` 
     - If marks info unavailable, estimates: MCQ = 1 mark lost, others = 2 marks
   - Creates a flaw record for each wrong answer
   - Appends to existing flawRecords, persists to IndexedDB
   - Skips sessions that couldn't be scored (no questions/answers array)

2. `getFlawSummary()` — returns aggregate stats:
   - Calls `aggregateFlaws(flawRecords, 7)` for sliding window
   - Returns `{ byCategory, bySeverity, topCategories, trend, totalFlaws, windowFlaws }`

3. `getFlawsBySkill(skill)` — filters flawRecords to a specific skill

4. `getCriticalFlaws(threshold = 'micro')` — returns flaws at or above severity threshold

5. `clearFlawData()` — resets all flaw records (for testing/reset)

6. `refresh(sessionList)` — re-processes all session data, detecting any new sessions since last run. Called on mount and can be called externally.

**Integration with existing sessions:**
- Accept sessions as a prop/argument (passed from App.jsx where useSkillAnalytics is instantiated)
- On mount: if sessions are available, run `processSessions(sessions)`
- Use a `processedSessionsRef` (useRef) to track which session IDs have been classified, avoiding re-processing
- On `refresh()` call, check for any unbounded session IDs and process them

**Persistence:**
- Load flawRecords from DSE_KEYS.FLAW_DATA on mount
- Save after each processSessions call
- Use silent try/catch for IndexedDB ops (existing pattern)

**Dependencies:**
- Import useIndexedDB from './useIndexedDB' (destructure getItem, setItem, DSE_KEYS)
- Import { classifyFlaw, aggregateFlaws } from '../utils/flawClassification'
- Import useCallback, useEffect, useState, useRef from 'react'

**Exports:**
```js
return {
  flawRecords, isLoaded,
  getFlawSummary, getFlawsBySkill, getCriticalFlaws,
  processSessions, refresh, clearFlawData,
  flawCount: flawRecords.length,
  lastProcessed: processedSessionsRef.current?.size || 0,
};
```

**Edge cases:**
- Empty sessions array → graceful no-op, flawRecords stays as loaded from IndexedDB
- Session with no answers → skip (can't classify without answer data)
- Session with questions but partial answers → classify what's available
- Session already processed → skip via processedSessionsRef Set
- Offline: entire system works offline (no network calls)
