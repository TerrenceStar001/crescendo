**Props:** `{ assessment, onComplete, onBack }` where `assessment` is the return value from `useAssessment()`.

1. `intro` — Welcome screen explaining the assessment, with "Start Assessment" button
2. `self-rate` — Grid of 4 skill cards (Reading, Writing, Listening, Speaking), each with 5 radio buttons for DSE level 1-5** (visualized as 1,2,3,4,5,5*,5**) and confidence selector (low/medium/high). Shows all 4 skills simultaneously for efficient UX. Next button proceeds when all 4 are rated.
3. `quiz` — Shows one skill at a time. For current skill, shows 3 MCQ questions (from `assessment.getQuizQuestions(skill)`). Student answers each, then clicks "Next Skill" or "Submit". Progress indicator: "Skill 2/4" 
4. `results` — Comparison table showing: self-rated level vs quiz-adjusted level vs final level for each skill. Visual indicator (✓ green for match, ⇅ yellow for adjusted, ! red for different). "Complete Assessment" button calls `assessment.completeAssessment()` then transitions to done.
5. `done` — Congratulations screen with final DSE levels shown as skill rings (mimicking Dashboard's SkillTile layout). "View My Study Plan" button calls `onComplete()`. "Re-take Assessment" calls `assessment.resetAssessment()` and returns to intro.

**Styling:**
- Clean, DSE-themed cards with skill icons 📖✍️🎧🎤
- CSS class prefix: `.assessment__` for BEM naming
- Follow existing card patterns from Dashboard.jsx (dashboard__section, dashboard__grid)
- Add CSS to App.css with section comment `/* ===== Assessment Page ===== */`
- Skill level selectors: vertical radio group with labels (1, 2, 3, 4, 5, 5*, 5**)

**Behavior:**
- MCQ options rendered as clickable cards (highlight selected)
- After selecting all 3 answers for a skill, auto-advance to next skill with brief (1.5s) confirmation
- Results phase: use a simple table layout showing self vs quiz-adjusted vs final
- All phases accessible via keyboard (tab through options, Enter to select)
- Each phase has a "Back" button except 'intro'
- Offline: entire component works without AI (quiz questions are hardcoded in useAssessment)
- Use `React.memo` per performance pattern for heavy components

**File export:** `export default function AssessmentPage({ assessment, onComplete, onBack })`
