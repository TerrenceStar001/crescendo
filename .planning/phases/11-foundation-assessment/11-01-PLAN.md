<task type="auto">
  <name>Task 1: Create flaw classification utility</name>
  <files>src/utils/flawClassification.js</files>
  <action>
Create `src/utils/flawClassification.js` with named exports:

1. `FLAW_CATEGORIES` — exported constant object with slugs, labels, and descriptions for 6 cognitive categories:
   - `vocabulary` — Wrong word choice, collocation errors, inappropriate register
   - `grammar` — Tense errors, subject-verb agreement, article misuse, preposition errors
   - `comprehension` — Failed to understand passage/question, selected wrong detail
   - `inference` — Drew unsupported conclusion, missed implicit meaning
   - `synthesis` — Failed to connect multiple ideas, missing comparative analysis
   - `expression` — Unclear phrasing, awkward structure, coherence gaps

2. `FLAW_SEVERITY` — exported constant with 3 levels:
   - `micro` — Single word/phrase error (1 mark lost, localized)
   - `meso` — Sentence/idea-level error (2-3 marks lost, affects partial understanding)
   - `macro` — Structural/conceptual error (4+ marks lost, fundamentally wrong approach)

3. `classifyFlaw(questionType, errorContext, marksLost)` — exported function that:
   - Takes question type (mcq/tfng/gap-fill/short-answer/matching/open-ended etc.), error context string, and marks lost
   - Returns `{ category: string, severity: string, confidence: number }`
   - Maps question types to categories: mcq/tfng→comprehension, gap-fill/short-answer→vocabulary, open-ended→expression/synthesis
   - Severity: marksLost >=4 → macro, 2-3 → meso, <=1 → micro
   - Uses keyword matching on errorContext for finer classification (e.g. "tense"→grammar, "infer"→inference, "word choice"→vocabulary)

4. `aggregateFlaws(flawRecords, windowDays = 7)` — exported function that:
   - Takes array of flaw records `{ category, severity, timestamp, skill, sessionId }`
   - Filters to within `windowDays` sliding window
   - Returns `{ byCategory: { category: count }, bySeverity: { severity: count }, topCategories: string[], trend: 'improving'|'declining'|'stable' }`
   - Trend: compare first half vs second half of window, if errors decreasing → improving, increasing → declining

5. Use named exports (not default). No React dependency — pure JS module.
6. No Try/catch blocks for logic errors — let callers handle.
7. Each function JSDoc-style comment explaining I/O.
</action>
