**Data model — Study Plan shape (stored at DSE_KEYS.STUDY_PLAN):**
```js
{
  _initialized: true,
  generatedAt: 'ISO string',
  density: { daysRemaining: number, density: 'low'|'medium'|'high'|'critical', sessionsPerWeek: number, dailyMinutes: number },
  tiers: {
    shortTerm: {
      label: 'This Week',
      span: '7 days',
      sessionsPerWeek: number,
      exercises: [
        { id: string, skill: string, constraints: { difficulty, type, timeLimit, theme, format, focus }, description: string, completed: boolean }
      ],
      focus: string,
    },
    midTerm: {
      label: 'This Month',
      span: '30 days',
      sessionsPerWeek: number,
      exercises: [
        { id: string, skill: string, constraints: { difficulty, type, timeLimit, theme, format, focus }, description: string, completed: boolean }
      ],
      focus: string,
    },
    longTerm: {
      label: 'Until Exam',
      span: 'X days',
      sessionsPerWeek: number,
      exercises: [
        { id: string, skill: string, constraints: { difficulty, type, timeLimit, theme, format, focus }, description: string, completed: boolean }
      ],
      focus: string,
    },
  },
  sourceAssessment: { assessmentDate: 'ISO string', levels: { reading: 1-5, writing: 1-5, listening: 1-5, speaking: 1-5 } },
  sourceFlaws: { topCategories: string[], severityBreakdown: object, updatedAt: 'ISO string' },
  version: number,
}
```

**State:**
- `plan` — current study plan object or null
- `isLoaded` — boolean
- `isGenerating` — boolean (true during AI generation)
- `error` — string | null

**Key functions (all useCallback):**

1. `loadPlan()` — called from useEffect on mount, loads plan from DSE_KEYS.STUDY_PLAN
2. `generatePlan(assessmentProfile, flawSummary, callAI)` — PLAN-01 core:
   - Takes assessment data + flaw summary + callAI function
   - Calls `calcDensity()` to determine plan density
   - Builds AI prompt (see below) and sends via `callAI`
   - Parses JSON response using same pattern as useDSEPapers.js (parseJSONObject)
   - Stores result to IndexedDB
   - Returns generated plan object

3. **AI Prompt for plan generation:**
   Use the same prompt pattern as useDSEPapers.js — system + user prompt, parseJSONObject:
   ```
   System: "You are a DSE English tutor creating personalized study plans. Return ONLY valid JSON."
   
   Prompt includes:
   - Student's current assessment levels for all 4 skills
   - Top flaw categories from flaw detection
   - Plan density (sessions/week, daily minutes)
   - Days remaining until DSE exam
   - Request for 3-tier plan with 3-5 exercises per tier
   - Each exercise has: skill, description, constraints {difficulty, type, timeLimit, theme, format, focus}
   - Temperature: 0.5 (creative but structured)
   - Max tokens: 3000
   - Timeout: 60000
   ```

4. `regeneratePlan(assessmentProfile, flawSummary, callAI)` — PLAN-04:
   - Increments version number
   - Calls generatePlan with same arguments (replaces existing plan)
   - Use try/catch: if AI fails, keep existing plan and set error state

5. `adaptPlan(newAssessment, newFlawSummary, callAI)` — PLAN-05:
   - Checks if assessment or flaw data has changed significantly (different from sourceAssessment/sourceFlaws)
   - If change detected, calls regeneratePlan
   - If no significant change, returns existing plan with no-op

6. `updateExerciseCompleted(exerciseId, tier, completed)` — Mark exercise as done
   - Finds exercise in plan.tiers[tier].exercises
   - Sets completed boolean
   - Persists updated plan

7. `getPlanByTier(tierName)` — returns exercises for a specific tier

8. `exportPlan()` — returns plan as formatted text (for sharing/copying)

9. `clearPlan()` — resets plan to null in IndexedDB

**Fallback (AI failure):** If AI generation fails or returns invalid JSON:
- Generate a default plan using density-only logic (no AI):
  - shortTerm: 5 generic exercises across skills based on density
  - midTerm: 10 exercises
  - longTerm: fill remaining days
  - Use buildConstraints() for exercise constraints
  - Set a `_isFallback: true` flag on the plan
  - Set error state with message about AI unavailability

**Persistence:**
- Load from DSE_KEYS.STUDY_PLAN in useEffect on mount
- Persist after generatePlan, regeneratePlan, adaptPlan, updateExerciseCompleted
- Single JSON blob per key pattern (same as useSkillAnalytics)

**Dependencies:**
- Import useIndexedDB from './useIndexedDB'
- Import { calcDensity, buildConstraints } from '../utils/planConstraints'
- Import { DSE_EXAM_DATE } from '../utils/planConstraints'
- Import useCallback, useEffect, useState, useRef from 'react'

**Exports:**
```js
return {
  plan, isLoaded, isGenerating, error,
  loadPlan, generatePlan, regeneratePlan, adaptPlan,
  updateExerciseCompleted, getPlanByTier, exportPlan, clearPlan,
  hasPlan: plan !== null,
};
```

**Edge cases:**
- No assessment data available → use default levels (all level 3)
- No flaw data available → skip flaw section in prompt
- AI call fails → fallback to default plan
- Multiple rapid regenerate calls → ignore if isGenerating is already true
- Plan version increments on each regeneration (for UI diff if needed)
